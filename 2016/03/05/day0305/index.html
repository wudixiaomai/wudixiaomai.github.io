###向客户端（浏览器）输出中文乱码的问题 ，resp.getOutputStream()用法问题

String data = "中文";
resp.getOutputStream().write(data.getBytes());
这个并不会乱码。 
String data = "中文";
resp.getOutputStream().write(data.getBytes("UTF-8"));
这个会乱码，为什么呢？ 中文字体变成字节，查的是UTF8码表，再以流的形式输出给浏览器时，浏览器接收到流数据，默认以GB2312编码显示数据，所以造成乱码。 >解决方式 解决中文乱码问题，通知浏览器以UTF-8的形式显示数据 1. resp.setHeader("content-type", "text/html;charset=utf-8"); //不要写错，写错了会下载的。 2. resp.setContentType("text/html;charset=utf-8"); 3. 模拟一个http响应头 
resp.getOutputStream().write("".getBytes()); meta标签显示问题请看，markdown文件信息
resp.getOutputStream().write("中国".getBytes("UTF-8"));
注意：不同浏览器效果不同，IE11可以正常解析，谷歌将数据当做文本显示。 ###向客户端（浏览器）输出数字的问题 **resp.getOutputStream()用法问题** 
 
req.getOutputStream().write(50);//不能正常显示
req.getOutputStream().write((50+"").getBytes());

###文件下载及中文文件下载方式 


public void doGet(HttpServletRequest req, HttpServletResponse resp) throws ServletException, IOException {
    //这只是转发请求到图片资源
    //RequestDispatcher rd = this.getServletContext().getRequestDispatcher(“/down/18岁的天空.jpg”);
    //rd.forward(req, resp);
    //1.获取请求资源的文件名（不包含实际路径）
    String realPath = this.getServletContext().getRealPath(“/download/18岁的天空.jpg”);
    //realPath = E:\Java\web\apache-tomcat-8.0.30\webapps\day06\download\18岁的天空.jpg
    String filename = realPath.substring(realPath.lastIndexOf(“\“)+1);

///2.设置http响应头（下载头），filename要给定URL编码,不然下载时文件名乱码。
resp.setHeader("Content-Disposition", "attachment;filename="+URLEncoder.encode(filename, "utf-8"));
//3.关联图片文件，读写操作
FileInputStream in = new FileInputStream(realPath);
System.out.println(in);
byte[] buffer = new byte[1042];
int len=0;
while((len=in.read(buffer))>0){
    resp.getOutputStream().write(buffer, 0, len);
}
if(in!=null){
    in.close();
}

}
